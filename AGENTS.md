# AGENTS.md

## Session Startup

When starting work in this repository, read these files first to recover project context before making changes:

1. `docs/command-center-design.md`
2. `docs/progress.md`

Use them as the source of truth for:

- current product direction
- architecture decisions
- implementation status
- next planned steps

## Working Rules

- Keep `docs/progress.md` updated when meaningful progress is made.
- Update `docs/command-center-design.md` when product or architecture decisions change.
- Prefer consistency with the approved design unless the user explicitly changes direction.
- When adding implementation, make sure new work aligns with the GitHub-to-VPS publishing model documented in the design doc unless the user decides otherwise.

## Documentation Expectations

- Record meaningful completed work in `docs/progress.md`.
- Record newly discovered TODOs or open questions in `docs/progress.md` if they would help future context recovery.
- Avoid letting the docs drift from the actual implementation.
